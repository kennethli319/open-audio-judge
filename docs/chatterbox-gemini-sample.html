Open Audio Judge Sample

Chatterbox TTS generation scored by the Gemini audio judge.

What This Demonstrates

This sample shows the TTS-first AutoJudge workflow: build or load TTS cases, synthesize audio with a local open model, send the generated audio plus target text to Gemini, and review an aggregate report with per-sample diagnostics.

1. Cases
JSONL cases with target text, turns, and metadata.
2. Generator
Chatterbox writes local audio files under runs/.
3. Judge
Gemini listens to audio and returns strict JSON scores.
4. Report
HTML and JSONL summarize issues, scores, and provenance.

How To Run It

Use this command for a small safe demo case. Source GEMINI_API_KEY at runtime; do not commit it to the repository. 

oaj autojudge-local-tts \
  --model mlx-community/chatterbox-turbo-6bit \
  --cases examples/tts_multiturn_cases.jsonl \
  --judge-provider gemini \
  --limit 1 \
  --out runs/chatterbox-tts-autojudge-smoke

If the local console script is stale while developing from source, run the same command through the module entry point: 

PYTHONPATH=src:. .venv/bin/python -m open_audio_judge.cli autojudge-local-tts \
  --model mlx-community/chatterbox-turbo-6bit \
  --cases examples/tts_multiturn_cases.jsonl \
  --judge-provider gemini \
  --limit 1 \
  --out runs/chatterbox-tts-autojudge-smoke

Expected Output Files

Path Purpose
synthesis/tts_audio_cases.jsonl Generated-audio cases with local audio_path, target text, and synthesis metadata.
model_summary.json Attempted/synthesized counts, success rate, audio totals, model/voice/language breakdowns, and failure counts.
judge-report/results.jsonl One strict JSON result per judged sample: score, reason, categories, notes, status, and provenance.
judge-report/report.html Human-readable report with aggregate scores, top issues, priority cases, and sample-by-sample details.

How To Read The Report

Start with average score and label counts, then inspect score breakdowns by TTS slice, model, voice, and language. Use the top diagnostics to find repeated issue clusters such as pronunciation errors, artifact classes, or weak slices. Finally, open the per-sample rows to inspect the exact Gemini reason, target text hash, generated-audio provenance, and any researcher notes.

This page is a committed sample artifact. Real run outputs should stay under ignored runs/ directories unless the audio, text, and metadata are explicitly safe to publish.
